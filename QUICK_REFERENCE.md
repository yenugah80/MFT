# MyFoodTracker: Behavioral Health Intelligence
## Developer Quick Reference

---

## 🎯 SYSTEM IN 60 SECONDS

**Goal**: Show personalized health recommendations without overwhelming users with raw data.

**How**:
1. Extract signals from food/mood/water logs
2. Detect correlations using rules (high-NOVA → mood crash, low hydration → fatigue, etc.)
3. Score correlations (strength 0-1, confidence 0-1, impact severity)
4. Filter by user's lifecycle stage (DISCOVERER → ELITE)
5. Decide what to show (SPEAK new pattern, REINFORCE good habit, PREDICT future, or SILENT)
6. Generate human-readable message with actions
7. Display beautifully on dashboard (only)

**Result**: "Mood dips 2-4h after high-processed meals. Try adding protein." (with visual gauge, 78% confidence)

## 🔧 KEY FUNCTIONS

### Backend

**Signal Extraction**
```javascript
extractFoodSignals(foodLog)      // → { nova, sugar, macros, cuisine... }
extractMoodSignals(moodLog)      // → { valence, arousal, energy, tags... }
extractWaterSignals(waterLog)    // → { hydration, type, time... }
```

**Correlation Detection**
```javascript
detectHighNovaMoodCrash(food, mood)    // → correlation or null
detectDehydrationFatigue(water, mood)  // → correlation or null
detectStressEatingPattern(...)         // → correlation or null
```

**Scoring**
```javascript
classifyConfoundingFactors(signals)    // → { factors, penalty, amplifier }
calculateConfidence(occurrences, base, bonus, confound)  // → 0-1
```

**Orchestration**
```javascript
determineLifecycleStage(userMetrics)   // → { stage, minConf, windows... }
```

## 🔗 API ENDPOINTS

### Correlations
```
POST   /api/correlations/compute
       └─ Compute correlations manually
       ├─ Body: { windowTypes: ['4h', '24h', ...], forceRecompute: false }
       └─ Response: { success, correlationCount, correlations: [...] }

GET    /api/correlations
       └─ Fetch user's active correlations
       ├─ Query: minConfidence=0.5, correlationType=mood_food, limit=10
       └─ Response: { success, correlationCount, correlations: [...] }

GET    /api/correlations/:id/evidence
       └─ Get evidence points for correlation
       └─ Response: { success, evidenceCount, evidence: [...] }

DELETE /api/correlations/:id
       └─ Mark correlation inactive
       └─ Response: { success, message }
```

### Orchestrator
```
POST   /api/orchestrator/run
       └─ Get today's recommendation for authenticated user
       └─ Response: {
         success,
         lifecycleStage: "TRACKER",
         decision: "SPEAK",
         message: { headline, subtitle, actions, visual... },
         correlations: [...]
       }

POST   /api/orchestrator/batch
       └─ Process all users (admin/cron only)
       ├─ Query: limit=100, batchSize=100
       └─ Response: { success, successCount, errorCount, results }
```

## 🚦 DECISION LOGIC

### SPEAK
```
Conditions:
  confidence >= 0.80
  occurrences <= 3
  healthImpactSeverity != 'positive'

Message: "New Pattern Found"
Style: "Here's something important we noticed..."
```

### REINFORCE
```
Conditions:
  confidence >= 0.70
  occurrences > 3
  OR healthImpactSeverity == 'positive'

Message: "Keep This Up" or "Remember The Pattern"
Style: "Your [domain] stays strong when you [behavior]"
```

### PREDICT
```
Conditions:
  lifecycleStage in [MASTER, CHAMPION, ELITE]
  confidence >= stage.minConfidence

Message: "Anticipatory Insight"
Style: "Based on your patterns, tomorrow likely..."
```

### SILENT
```
Conditions:
  No correlations above threshold
  OR user is optimal (no action needed)

Message: "You're Optimized"
Style: "Your patterns are stable. Keep logging to discover."
```

## 🐛 DEBUGGING TIPS

### Issue: Confidence too low?
1. Check confounding factors: sleep, stress, exercise?
2. Verify occurrence count: need >= 3 for good confidence
3. Check consistency bonus applied: `(1 + bonus)` not `* bonus`

### Issue: Pattern not detected?
1. Verify logs exist for both signals (food + mood)
2. Check time lag matches rule (e.g., 4h window for food-mood)
3. Ensure signal extraction produces expected values
4. Log intermediate values: `console.log({ foodSignals, moodSignals })`

### Issue: Recommendation looks wrong?
1. Verify it came from `generateRecommendationObject()`, not direct correlation
2. Check lifecycle stage: DISCOVERER shows nothing, BUILDER shows 1, etc.
3. Ensure message uses Recommendation schema, not Correlation fields

### Issue: UI doesn't render?
1. Verify API returns data: `POST /api/orchestrator/run`
2. Check data structure matches `Recommendation` schema
3. Verify no correlation IDs leaked into UI
4. Test with mock data if API fails

## ⚡ PERFORMANCE TIPS

### Queries
- Index: `user_correlations(userId, isActive, confidence DESC)`
- Fetch only: `limit = lifecycleStage.correlationsToShow + 2`
- Cache orchestration result: no recompute if called twice same day

### Batch Processing
- Process users in batches of 100 (parallelizable)
- Run at 2am UTC (low traffic)
- Skip users with `lastLogDate > 30 days ago`

### Frontend
- Memoize components: `React.memo(PatternGauge)`
- Lazy load detail sheet only on expand
- Cache recommendation result in React Query: `staleTime: 24h`
